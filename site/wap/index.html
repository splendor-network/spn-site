Splendor Network Technology Advantages

Multi-Chain

Splendor Network smart contracts support multi-chain. Through the Jury consensus and adaptation layer, Splendor Network smart contracts can run on different blockchains at the same time. Splendor Network smart contract can interact with diﬀerent blockchains at the same time, so users can trade tokens from diﬀerent blockchains in one Splendor Network contract invocation to make the inter-chain token exchange distributed, atomic, and immutable.

Multi-Task

The verification, execution, etc. of the Splendor Network smart contract can be done by the Jury specifically created for it. The Jury is composed of a group of jurors at random selected from the candidate jurors randomly. Smart contracts in Splendor Network can be implemented by selecting different juries in a multi-tasking manner. Compared with the whole-network consensus approach, Splendor Network network congestion will be effectively reduced.

Security

Splendor Network's security is reflected in two aspects, one is the security of development, and the other is the security of contract execution. When a user develops a corresponding scene contract, he/she only needs to call a standard contract and complete DAPP development in a few steps. Splendor Network VM can provide a safe and efficient sandbox operating environment for smart contracts.

Multi-Language & Multi-Platform

· DApp developers of Splendor Network can develop smart contracts using mainstream development languages without having to learn a new contract development language.

· Splendor Network VM is completely decoupled from the underlying operation system, which allow Splendor Network smart contract adapt to windows, Linux, Mac as well as other types of platform.

Q&A

Differences with other inter-chain solution?

Most recent solutions deal with blockchain interoperability by building one trustable adapter for each pair of chains. One popular adapter can be the inter-chain that consists of fully replicated nodes running the same consensus protocol. In comparison, Splendor Network bridges one chain with others through a decoupled, lightweight, and meta-level smart contract layer. The consensus is reached among the Jurors selected before the contract execution. Jurors responsible for execution have the contract states.

Which blockchain does Splendor Network support?

Many new blockchains are forked from Bitcoin and Ethereum. We are starting with Bitcoin and Ethereum, and then move to other popular blockchains as soon as possible. We encourage community developers to contribute to the bridge with other chains.

What do you mean by decoupling contract?

Decoupling Contract means that the contract execution is decoupled from the blockchain beneath and is done at Splendor Network level. After execution, contract states are stored in the Jury and/or the blockchain beneath.
