Technology Advantages

Cross-Chain

Splendor Network integrates with other Blockchains by utilizing a hybrid two-way peg mechanism, in which transaction verification is achieved by using merkle-patricia-proof. BFT+DPoS is used to for selecting Validators and coming at consensus on transactions and state changes.

Multi-Channel

Smart contracts on the Splendor Network can be executed and verified by a selected group of Validator. Different Smart contracts on Splendor Network can choose different groups of Validators, providing a Multi-Channel consensus mechanism with faster consensus reaching.

Developer-Friendly

DApp developers of Splendor Network can do programs with mainstream development languages such as Go and Java, avoiding a steep learning curve. Splendor Network VM is docker based and could be set up and running on various operating systems, maximiazing the reusability of existing IT infrastructure.
